本文档记录 AtomBox Desktop 第一版应用设置页设计。本文档只定义结构和交互，不定义具体颜色、字体、间距或图标尺寸。

## 1. 页面定位

应用设置页用于管理 AtomBox 的全局偏好。

进入方式：

- 点击左侧菜单 `设置 / 应用设置`。

应用设置页只放全局应用行为，不放账号配置，不放远程文件操作，不放诊断能力。

## 2. 页面线框图

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 应用设置                                                                     │
├──────────────────────────────────────────────────────────────────────────────┤
│ 启动后默认页面       [ 首页 ▼ ]                                              │
│ 关闭窗口时           [ 关闭应用 / 最小化应用 ▼ ]                              │
│ 默认下载目录         [ D:\Downloads\AtomBox                  ][选择]          │
│ 最大并发数           [ 3 ▲▼ ]                                                │
│ 配置目录             [打开]                                                  │
│ 日志目录             [打开]                                                  │
│ 状态目录             [打开]                                                  │
│                                                                              │
│                                                         [重置] [保存]        │
└──────────────────────────────────────────────────────────────────────────────┘
```

第一版设置页采用一组纵向设置行。左列为设置项名称，右列为对应控件。

## 3. 设置项

| 设置项 | 说明 |
| --- | --- |
| 启动后默认页面 | 应用启动后默认进入的主页面。默认值为“首页”。 |
| 关闭窗口时 | 控制关闭主窗口时退出应用或最小化到系统托盘。 |
| 默认下载目录 | 下载任务默认保存位置。 |
| 最大并发数 | Transfer Runtime 的默认并发任务数量。 |
| 配置目录 | 打开本地配置目录。 |
| 日志目录 | 打开本地日志目录。 |
| 状态目录 | 打开本地状态目录。 |

缓存相关 UI 第一版暂时隐藏。缓存目录和清理缓存不作为当前发布基线的可见设置项。

## 4. 操作

页面右下角提供：

- 重置。
- 保存。

按钮规则：

| 按钮 | 行为 |
| --- | --- |
| 保存 | 保存当前设置，停留在应用设置页。只有页面 dirty 时启用。 |
| 重置 | 恢复为当前持久化设置或默认设置。 |

第一版不做自动保存。所有设置修改必须点击“保存”后才生效或持久化。

## 5. 字段交互

| 字段 | 行为 |
| --- | --- |
| 启动后默认页面 | 下拉选择，包含首页、远程存储、传输队列、账号管理等入口。 |
| 关闭窗口时 | 下拉选择：关闭应用、最小化应用。 |
| 默认下载目录 | 只读输入框展示路径，点击右侧按钮打开系统目录选择弹窗。 |
| 最大并发数 | 整数数字输入，不能变成浮点数。 |
| 配置目录 | 点击按钮打开本地配置目录。 |
| 日志目录 | 点击按钮打开本地日志目录。 |
| 状态目录 | 点击按钮打开本地状态目录。 |

关闭窗口时选择“最小化应用”后，关闭主窗口应隐藏到系统托盘，托盘右键菜单提供显示和退出入口。

## 6. 状态来源

应用设置页展示的是 Application 暴露的应用设置结果。

约束：

- ViewModel 不直接读写配置文件。
- ViewModel 不直接访问 Infrastructure。
- ViewModel 不直接修改 Transfer Runtime。
- 设置读取和保存必须通过 Application 用例进入系统。

## 7. AtomUI 使用

| 场景 | AtomUI 组件 |
| --- | --- |
| 下拉选择 | `ComboBox` |
| 只读路径 | `LineEdit` |
| 整数并发 | `NumberUpDown` |
| 保存 / 重置 / 打开目录 | `Button` |
