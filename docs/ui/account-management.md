本文档记录 AtomBox Desktop 第一版账号管理页设计。本文档只定义结构和交互，不定义具体颜色、字体、间距或图标尺寸。

## 1. 页面定位

账号管理页用于集中查看、编辑、删除远程连接账号。

进入方式：

- 点击左侧菜单 `设置 / 账号管理`。

账号管理页不是远程浏览入口，不展示远程文件列表，不展示 secret 明文。

## 3. 账号列表

账号列表采用行列表。

固定列：

| 列 | 内容 |
| --- | --- |
| 名称 | 账号显示名称。 |
| 类型 | OSS、FTP/SFTP、WebDAV、网盘等 provider category。 |
| 地址 / Provider | OSS 显示 provider + region 或 endpoint；FTP/SFTP 显示 host:port。 |
| 操作 | 编辑、删除。 |

账号列表不展示“状态”列。连接是否可用由账号弹窗中的连接测试或远程浏览实际请求反馈，不在账号管理列表中长期维护。

## 4. 操作

| 操作 | 行为 |
| --- | --- |
| 添加账号 | 通过右下角 AtomUI `FloatButton` 打开统一账号弹窗。 |
| 编辑 | 打开统一账号弹窗，进入编辑模式。 |
| 删除 | 弹出确认弹窗；确认后删除账号。 |

新增和编辑必须复用 `docs/ui/account-dialog.md` 定义的账号弹窗。编辑模式不允许修改连接类型。

删除必须使用确认弹窗，并检查未完成传输任务引用。

## 5. 行交互

| 行为 | 规则 |
| --- | --- |
| 鼠标悬停 | 当前行背景变色，与文件列表、传输列表保持一致。 |
| 单击行 | 第一版不定义额外行为。 |
| 双击行 | 第一版不定义默认行为。 |
| 右键行 | 第一版不定义右键菜单。 |

## 6. 空状态

当没有账号时，右侧 Content Area 显示空状态，并提供添加账号入口。

空状态中的新增入口也必须打开统一账号弹窗。

## 7. 状态来源

账号管理页展示的是 Application 暴露的账号列表结果。

约束：

- ViewModel 不直接访问 credential store。
- ViewModel 不直接调用 Provider。
- ViewModel 不直接测试连接。
- 刷新账号列表不等于测试连接。

## 8. AtomUI 使用

| 场景 | AtomUI 组件 |
| --- | --- |
| 添加账号 | `FloatButton` |
| 编辑、删除 | `Button` |
| 删除确认 | `Modal` |
| 空状态 | `Empty` |
