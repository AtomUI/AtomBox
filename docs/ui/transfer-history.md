# AtomBox Desktop 传输历史页

> 文档状态：第一版发布基线冻结
>
> 冻结时间：2026-06-22
>
> 冻结范围：当前文档定义的传输历史页布局、历史列表结构、分页、清理历史和行交互
>
> 变更规则：第一版发布前代码已冻结；如需调整本文档定义的传输历史页布局或交互边界，必须先说明原因，再同步更新相关 UI 文档。

本文档记录 AtomBox Desktop 第一版传输历史页设计。本文档只定义结构和交互，不定义具体颜色、字体、间距或图标尺寸。

## 1. 页面定位

传输历史页用于展示已经结束的传输任务记录。

进入方式：

- 点击左侧菜单 `传输 / 历史记录`。

第一版传输历史页只展示已完成、失败、取消或中断的历史任务，不展示当前运行队列。当前任务进入独立传输队列页。


第一版不提供筛选条、不提供导出记录。

## 3. 历史列表

历史列表采用只读行列表。

固定列：

| 列 | 内容 |
| --- | --- |
| 文件名 | 传输对象名。 |
| 方向 | 上传或下载。 |
| 目标 / 来源 | 上传时展示远程目标；下载时展示本地目标或远程来源摘要。 |
| 结果 | 成功、失败、取消或中断。 |
| 完成时间 | 任务结束时间。 |

历史列表第一版不做分组、不做多选、不做批量选中。

## 4. 分页器

分页器固定显示在传输历史页底部。

第一版分页器只包含：

- 上一页。
- 下一页。

分页规则：

- 默认每页显示固定数量历史记录。
- 无上一页时，“上一页”不可用。
- 无下一页时，“下一页”不可用。
- 不显示页码。
- 不支持跳页。
- 不支持每页条数选择。
- 当前页历史记录数量显示在底部左侧。

## 5. 清理历史

- 右上角不提供文字按钮。
- 右下角使用 AtomUI `FloatButton` 提供清理历史入口。
- 清理历史必须二次确认。
- 确认文案必须说明清理范围。
- 清理失败时显示错误摘要，并提供错误详情入口。

## 6. 行交互

| 行为 | 规则 |
| --- | --- |
| 鼠标悬停 | 当前行背景变色。 |
| 单击行 | 当前行选中，背景固定显示。 |
| 双击行 | 第一版不定义默认行为。 |
| 右键行 | 第一版不定义右键菜单。 |

历史记录路径列可以在文本过长时使用 AtomUI `Tooltip` 展示完整路径；Tooltip 仅用于历史记录页，不用于传输队列页。

## 7. 空状态

当没有历史记录时，右侧 Content Area 显示空状态。

空状态不提供新增任务按钮。传输任务必须从远程浏览页或后续明确入口创建。

## 8. 状态来源

传输历史页展示的是 Application 暴露的历史记录查询结果。

约束：

- ViewModel 不直接访问 TransferTaskStore。
- ViewModel 不直接访问 Transfer Runtime。
- ViewModel 不直接调用 Infrastructure。
- 历史记录必须通过 Application 用例或查询结果进入 Presentation。

## 9. AtomUI 使用

| 场景 | AtomUI 组件 |
| --- | --- |
| 清理历史 | `FloatButton` |
| 清理确认 | `Modal` |
| 长路径提示 | `Tooltip` |
| 消息提示 | `Message` |
