# AtomBox Desktop 账号弹窗

> 文档状态：第一版发布基线冻结
>
> 冻结时间：2026-06-22
>
> 冻结范围：当前文档定义的账号添加/编辑弹窗布局、字段分组、连接测试、SFTP/FTP/OSS/WebDAV/网盘字段和保存规则
>
> 变更规则：第一版发布前代码已冻结；如需调整本文档定义的账号弹窗布局或交互边界，必须先说明原因，再同步更新相关 UI 文档。

本文档记录 AtomBox Desktop 第一版账号添加/编辑弹窗设计。本文档只定义结构和交互，不定义具体颜色、字体、间距或图标尺寸。

## 1. 弹窗定位

账号弹窗用于新增或编辑远程连接配置。

当前代码已注册的 provider 范围：

- 对象存储：阿里云 OSS、腾讯云 COS、七牛云 Kodo、又拍云 USS、华为云 OBS、百度智能云 BOS、京东云 OSS、青云 QingStor、火山引擎 TOS。
- 文件传输：SFTP、FTP、WebDAV。
- 网盘 API：阿里云盘、百度网盘 provider 代码与注册已存在，但第一版发布基线不把它们作为已充分验收的主路径能力。

当前账号弹窗可见范围与注册范围不同：京东云 OSS 和青云 QingStor 代码已注册，但在添加账号弹窗中暂时隐藏，等待完成发布级真实验收后再开放；百度智能云 BOS 已在账号弹窗中开放。

账号弹窗只属于 Desktop / Presentation 层。Application 不知道弹窗存在，Core、Transfer、Providers、Infrastructure 不依赖弹窗概念。

第一版采用统一弹窗骨架，根据连接类型和 provider 切换字段。

## 2. 进入规则

| 入口 | 弹窗行为 |
| --- | --- |
| 远程存储汇总页右下角添加按钮 | 打开账号弹窗，标题为“添加远程连接”，连接类型由用户选择。 |
| 远程浏览页无账号空状态 `添加账号` | 打开账号弹窗，标题为“添加远程连接”，连接类型可按当前分类预选。 |
| 账号管理页添加按钮 | 打开账号弹窗，标题为“添加远程连接”，连接类型由用户选择。 |
| 账号管理页编辑按钮 | 打开账号弹窗，标题为“编辑远程连接”，加载已有账号配置。 |

新增模式允许选择连接类型。编辑模式不允许修改连接类型；如果用户要更换类型，必须新建账号。

## 3. 统一弹窗骨架

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 添加远程连接                                                       [×]       │
├──────────────────────────────────────────────────────────────────────────────┤
│ [连接测试结果 MessageCard，默认占位不挤压主体布局]                           │
│                                                                              │
│ 连接类型                                                                     │
│ 类型             [ OSS ▼ ]                                                   │
│ Provider / 协议  [ Aliyun OSS ▼ ]                                            │
│                                                                              │
│ 基本信息                                                                     │
│ 显示名称         [                                      ]                    │
│                                                                              │
│ 连接配置                                                                     │
│ ... 根据连接类型切换字段 ...                                                 │
│                                                                              │
│ 凭据                                                                         │
│ ... 根据连接类型切换字段 ...                                                 │
│                                                                              │
├──────────────────────────────────────────────────────────────────────────────┤
│                                       [连接测试] [取消] [保存]               │
└──────────────────────────────────────────────────────────────────────────────┘
```

连接测试反馈使用 AtomUI `MessageCard`，显示时间为短时提示；它必须有稳定预留区域，不能在出现和消失时挤压弹窗主体造成跳动。

## 4. OSS 弹窗

OSS 第一版字段：

| 分区 | 字段 | 描述 |
| --- | --- | --- |
| 连接类型 | 类型、Provider / 协议 | 展示当前允许在账号弹窗中选择的对象存储 provider；京东云 OSS 和青云 QingStor 暂时隐藏。 |
| 基本信息 | 显示名称 | 用户自定义账号显示名。 |
| 连接配置 | Endpoint、Region、Bucket/Root 等 | 具体字段由 provider descriptor 决定。 |
| 凭据 | AccessKeyId、AccessKeySecret 等 | 不回显 secret 明文。 |

Bucket 可以为空。Bucket 为空时，进入该账号后默认展示 bucket 列表。

## 5. FTP 弹窗

FTP 第一版字段：

| 分区 | 字段 | 描述 |
| --- | --- | --- |
| 连接类型 | 类型、Provider / 协议 | FTP。 |
| 基本信息 | 显示名称 | 用户自定义账号显示名。 |
| 连接配置 | Host、Port、传输模式、超时等 | 支持域名或 IP。 |
| 凭据 | 匿名访问 / 用户名 + 密码 | 匿名访问启用时不要求用户名密码。 |

第一版 FTP 认证方式包含“匿名访问”和“用户名 + 密码”。FTPS/TLS 证书模式暂不作为第一版发布基线能力。

## 6. SFTP 弹窗

SFTP 第一版字段：

| 分区 | 字段 | 描述 |
| --- | --- | --- |
| 连接类型 | 类型、Provider / 协议 | SFTP。 |
| 基本信息 | 显示名称 | 用户自定义账号显示名。 |
| 连接配置 | Host、Port、Home | Host 支持域名或 IP；Home 可由连接测试成功后回填。 |
| 凭据 | 用户名、认证方式、密码或私钥 | 认证方式为“密码”时只显示密码输入；认证方式为“私钥”时只显示私钥输入。 |

连接测试成功后，如果 provider 返回 Linux home 路径，弹窗应回填 Home 字段。左侧选择该 SFTP provider 时，默认打开 Home 目录。

## 7. WebDAV 弹窗

WebDAV 第一版字段：

| 分区 | 字段 | 描述 |
| --- | --- | --- |
| 连接类型 | 类型、Provider / 协议 | WebDAV。 |
| 基本信息 | 显示名称 | 用户自定义账号显示名。 |
| 连接配置 | Endpoint / Root | WebDAV 服务地址和可选根路径。 |
| 凭据 | 匿名访问 / 用户名 + 密码 | 支持匿名 WebDAV 和 Basic 认证。 |

## 8. 连接测试

连接测试规则：

- 点击“连接测试”只验证当前表单配置，不保存账号。
- 测试过程中按钮进入不可重复点击状态。
- 测试成功后显示成功状态。
- 测试失败后显示错误摘要，错误详情通过错误详情弹窗展示。
- 测试失败不阻止保存。
- 测试结果是当前表单快照的结果；用户继续编辑字段后，测试状态应回到“未测试”或“需要重新测试”。

保存不强制要求先测试连接。离线编辑、临时保存、后续再修正配置都必须被允许。

## 9. 保存与取消

| 操作 | 行为 |
| --- | --- |
| 保存 | 保存账号配置，关闭弹窗，刷新左侧菜单。 |
| 取消 | 不保存任何修改，关闭弹窗。 |
| 右上角关闭 | 等同取消。 |
| 从远程浏览页无账号空状态保存成功 | 选中新账号，刷新左侧菜单，并打开该账号的远程浏览页。 |
| 从账号管理页新增保存成功 | 关闭弹窗，刷新账号管理页和左侧菜单，不强制跳转远程浏览页。 |

编辑模式保存成功后，应刷新左侧连接实例名称和右侧当前页面显示状态。

## 10. Secret 展示规则

Secret 字段包括 AccessKeySecret、密码、Token 等敏感内容。

第一版规则：

- 新增模式下，secret 字段为空，由用户输入。
- 编辑模式下，不回显 secret 明文。
- 编辑模式下，secret 字段显示“已配置”状态。
- 用户需要修改 secret 时，再重新输入新值。
- Secret 不写入日志，不进入普通配置文件，不进入错误详情。

| 弹窗 | `Dialog` / `Modal` |
