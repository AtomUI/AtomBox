# AtomBox Desktop 传输队列页

> 文档状态：第一版发布基线冻结
>
> 冻结时间：2026-06-22
>
> 冻结范围：当前文档定义的传输队列页布局、任务列表结构、取消/重试交互和状态展示
>
> 变更规则：第一版发布前代码已冻结；如需调整本文档定义的传输队列页布局或交互边界，必须先说明原因，再同步更新相关 UI 文档。

本文档记录 AtomBox Desktop 第一版传输队列页设计。本文档只定义结构和交互，不定义具体颜色、字体、间距或图标尺寸。

## 1. 页面定位

传输队列页用于展示当前传输任务和可操作的传输状态。

进入方式：

- 点击左侧菜单 `传输 / 传输队列`。
- 从远程浏览页上传或下载文件后自动跳转。

第一版传输队列页只展示当前队列，不混入历史记录。历史任务进入独立历史记录页。

## 2. 页面线框图

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 传输队列                                                                     │
├──────────────┬────────┬─────────────────────────────┬────────┬────────┬──────┤
│ 文件名       │ 方向   │ 目标 / 来源                 │ 进度   │ 速度   │ 操作 │
├──────────────┼────────┼─────────────────────────────┼────────┼────────┼──────┤
│ file.zip     │ 上传   │ oss://bucket/file.zip       │ 45%    │ 3.2M/s │ 取消 │
│ backup.tar   │ 上传   │ oss://bucket/backup.tar     │ 失败   │ -      │ 重试 │
│ video.mp4    │ 下载   │ D:\Downloads\video.mp4      │ 等待中 │ -      │ 取消 │
│              │        │                             │        │        │      │
│                                                                   [重试]     │
└──────────────┴────────┴─────────────────────────────┴────────┴────────┴──────┘
```

第一版不提供筛选条。上传、下载、运行中、失败等分类不进入页面顶部筛选，也不进入左侧菜单。

## 3. 任务列表

任务列表采用行列表。

固定列：

| 列 | 内容 |
| --- | --- |
| 文件名 | 传输任务的文件名或对象名。 |
| 方向 | 上传或下载。 |
| 目标 / 来源 | 上传时展示目标远程路径；下载时展示本地目标路径或远程来源摘要。 |
| 进度 | 百分比、失败、中断、取消或完成状态。 |
| 速度 | 当前传输速度；非运行状态显示 `-`。 |
| 操作 | 当前任务可执行的主要操作。 |

任务列表第一版不做分组、不做多选、不做批量选中。

传输队列页第一版不分页。当前队列是运行态视图，分页会隐藏正在变化的任务状态。

## 4. 单行操作

单行操作根据任务状态显示一个主要动作。

| 任务状态 | 操作 |
| --- | --- |
| 等待中 | 取消 |
| 运行中 | 取消 |
| 失败 | 重试 |
| 中断 | 重试 |
| 已完成 | - |
| 已取消 | - |

第一版不提供用户可见的暂停/继续功能。`Paused` 是传输模型中的保留状态，不作为第一版对外交互能力。

## 5. 批量操作

- 第一版不展示“暂停全部”“继续全部”“清理完成”文字按钮。
- 如果存在可重试任务，右下角使用 AtomUI `FloatButton` 提供“重试失败项”入口。
- 等待中批量取消入口暂不开放，避免“取消等待中”语义造成误解。

## 6. 行交互

| 行为 | 规则 |
| --- | --- |
| 鼠标悬停 | 当前行背景变色。 |
| 单击行 | 当前行选中，背景固定显示。 |
| 双击行 | 第一版不定义默认行为。 |
| 右键行 | 第一版不定义右键菜单。 |

第一版传输队列不做右键菜单，所有操作通过行内操作按钮或右下角 FloatButton 完成。

失败或中断行可以提供“详情”辅助入口。详情入口调用统一错误详情弹窗，不引入任务详情抽屉。

## 7. 空状态

当没有传输任务时，右侧 Content Area 显示空状态。

空状态不提供新增任务按钮。传输任务必须从远程浏览页或后续明确入口创建。

## 8. 状态来源

传输队列页展示的是 Application 暴露的传输状态结果。

约束：

- ViewModel 不直接订阅 Transfer Runtime。
- ViewModel 不直接访问 TransferQueue。
- ViewModel 不直接调用 Provider。
- 传输状态必须通过 Application 用例或查询结果进入 Presentation。

## 9. 收口规则

- 失败状态必须展示失败摘要。
- 中断状态必须与失败、取消区分。
- 用户取消的任务展示为取消，不展示为失败。
- 应用关闭、worker 超时或结果未知导致的停止展示为中断。
- 重试通过 `TransferAppService` 进入调度链路，UI 不直接操作 Transfer Runtime。
- 任务完成通知需要合并，避免批量任务完成时连续弹出大量 Message。

## 10. AtomUI 使用

| 场景 | AtomUI 组件 |
| --- | --- |
| 单行操作 | `Button` |
| 重试失败项 | `FloatButton` |
| 消息提示 | `Message` |
| 错误详情 | `Modal` / 错误详情弹窗服务 |
