本文档记录 AtomBox Desktop 第一版首页整体布局。本文档只定义低保真结构和区域职责，不定义具体颜色、字体、图标、间距或圆角。

## 1. 布局原则

AtomBox Desktop 第一版采用资源管理器式主界面：

- 顶部为系统窗口 TopBar，只保留窗口标题、软件图标和窗口控制按钮。
- 左侧为软件 Logo/名称区域和 Navigation Tree。
- 中间右侧为唯一 Content Area。
- 底部为常驻 StatusBar。
- 第一版不做多 Tab、多工作区或复杂 Docking。

新增或编辑 OSS、FTP、SFTP、WebDAV 等远程连接配置，统一通过弹窗完成，不塞进首页主内容区。

## 3. 区域职责

### 3.1 TopBar

TopBar 是窗口级标题栏。

允许包含：

- 软件图标。
- 产品名称。
- 最小化、最大化、关闭按钮。

禁止包含：

- 页面级操作按钮。
- 远程文件操作。
- 账号新增入口。

### 3.2 Navigation Tree

Navigation Tree 是左侧功能菜单。

第一版一级菜单如下：

- 首页。
- 远程存储。
- 传输。
- 设置。

远程存储下展示：

- OSS。
- FTP/SFTP。
- 各分类下真实账号节点。

左侧菜单的完整结构和点击行为记录在 `docs/ui/menu.md`。

### 3.3 Home Content

首页右侧内容区展示：

- AtomBox Logo 和软件名。
- 远程存储、传输队列、账号管理三个主要入口按钮。
- 账号数量摘要。
- 当前队列摘要。
- 最近完成历史摘要。
- 最近传输记录。

首页不是营销页，也不是设置页。它只作为进入主要工作流的轻量工作台。

### 3.4 Content Area

第一版所有主页面都在该区域显示：

- 首页。
- 远程浏览页面。
- 传输队列页面。
- 历史记录页面。
- 应用设置页面。
- 账号管理页面。

### 3.5 StatusBar

StatusBar 是底部常驻状态栏。

允许显示：

- 当前运行中传输数量。
- 上传速度。
- 下载速度。
- 错误数量。

StatusBar 只展示摘要，不承载完整传输管理。完整传输管理必须进入传输队列页面。

## 4. 弹窗规则

以下流程第一版必须通过弹窗完成：

- 新增远程连接。
- 编辑账号配置。
- 连接测试。
- 删除确认。
- 错误详情。

弹窗只属于 Desktop / Presentation 层。Application 不知道弹窗存在，Core、Transfer、Providers、Infrastructure 更不能依赖弹窗概念。

## 5. AtomUI 使用

| 场景 | AtomUI 组件 |
| --- | --- |
| 主窗口 | `Window` |
| 左侧菜单 | `NavMenu` |
| 首页入口按钮 | `Button` |
| 近期列表 | 行列表 + AtomUI 文本和按钮 |
| 消息提示 | `Message` |
