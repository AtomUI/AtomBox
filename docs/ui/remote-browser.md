# AtomBox Desktop 远程浏览页

> 文档状态：第一版发布基线冻结
>
> 冻结时间：2026-06-22
>
> 冻结范围：当前文档定义的远程存储汇总首页、分类入口页、具体 provider 文件浏览页、文件列表交互和第一版收口规则
>
> 变更规则：第一版发布前代码已冻结；如需调整本文档定义的远程浏览页布局或交互边界，必须先说明原因，再同步更新相关 UI 和 Presentation 文档。

本文档记录 AtomBox Desktop 第一版远程浏览页的低保真 UI 设计。本文档只定义结构和交互，不定义具体颜色、字体、间距或图标尺寸。

## 1. 页面定位

远程浏览页是右侧 Content Area 中的核心工作页面，用于展示：

- 远程存储汇总首页。
- 某一类远程连接的账号列表或空状态。
- 某个具体远程连接下的 bucket、目录或文件列表。

远程浏览页只有在加载远程文件列表时才必须绑定具体连接实例。远程存储汇总首页、分类账号列表和无账号空状态可以不绑定具体连接。

`OSS`、`FTP/SFTP`、`WebDAV` 只是资源类型分组，不是具体连接。没有具体账号、凭据和默认路径，就不能直接请求远程文件列表。

## 2. 进入规则

左侧菜单点击行为如下：

| 点击节点 | 右侧 Content Area 行为 |
| --- | --- |
| 远程存储 | 打开远程存储汇总首页，展示全部已添加账号，并按对象存储、FTP/SFTP、WebDAV 分组。 |
| OSS | 展示 OSS 分类账号列表或空状态，不自动打开唯一账号。 |
| FTP/SFTP | 展示 FTP/SFTP 分类账号列表或空状态，不自动打开唯一账号。 |
| WebDAV | 展示 WebDAV 分类账号列表或空状态，不自动打开唯一账号。 |
| 具体连接实例 | 打开远程浏览页，并加载该连接默认路径或上次路径。 |

默认路径规则：

- OSS：默认进入 bucket 列表。
- SFTP：优先进入账号配置中的 Home 路径；没有 Home 时进入根路径。
- FTP：进入远程根路径或账号配置对应默认路径。

## 3. 汇总首页

当用户进入 `远程存储` 总入口且已经存在账号时，右侧展示汇总首页。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 远程存储                                                                     │
├──────────────────────────────────────────────────────────────────────────────┤
│ 全部账号 3          对象存储 2          FTP/SFTP 1          WebDAV 1                            │
│                                                                              │
│ ┌ 对象存储 ────────────────────────────────────────────────────────────────┐ │
│ │ Aliyun Prod       [ALIYUN-OSS]       oss-cn-hangzhou...       [打开]     │ │
│ │ Tencent Test      [TENCENT-COS]      cos.ap-guangzhou...      [打开]     │ │
│ └──────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│ ┌ FTP / SFTP ──────────────────────────────────────────────────────────────┐ │
│ │ SFTP Dev          [SFTP]             sftp.example.com:22       [打开]    │ │
│ └──────────────────────────────────────────────────────────────────────────┘ │
│                                                                   [+]        │
└──────────────────────────────────────────────────────────────────────────────┘
```

汇总首页规则：

- 顶部使用 AtomUI `Statistic` 展示全部账号、对象存储、FTP/SFTP、WebDAV 数量。
- 分组容器使用 AtomUI `Card`。
- provider 类型使用 AtomUI `Tag`。
- 账号行支持双击打开，也提供 AtomUI `Button` 打开。
- 右下角使用 AtomUI `FloatButton` 添加账号。

当没有任何账号时，展示 AtomUI `Empty` 和 `添加账号` 主按钮；底部状态栏不展示额外引导文案。

## 4. OSS 文件浏览

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ [provider name]                                                              │
│ [bucket ▼] [bucket/folder breadcrumb]                 [按名称前缀搜索] [🔍] │
├────────────┬───────────────────────────────┬─────────────┬───────────────────┤
│ 缩略图     │ 文件名 / 桶名                 │ 文件大小    │ 修改时间          │
├────────────┼───────────────────────────────┼─────────────┼───────────────────┤
│ [box]      │ bucket-a                      │ -           │                   │
│ [folder]   │ folder-a                      │ -           │ 2026-06-13        │
│ [file]     │ file.zip                      │ 1.2 GB      │ 2026-06-13        │
├────────────┴───────────────────────────────┴─────────────┴───────────────────┤
│ 当前目录项目 3 个                                      [上一页]    [下一页] │
│                                                                   [刷新]     │
│                                                                   [上传]     │
│                                                                   [新建夹]   │
└──────────────────────────────────────────────────────────────────────────────┘
```

OSS 规则：

- 未选择 bucket 时，文件列表区域展示 bucket 列表，列名为“桶名”。
- 选择或双击 bucket 后，列表展示该 bucket 内的文件和文件夹，列名改为“文件名”。
- bucket 选择使用 AtomUI `ComboBox`。
- 路径导航使用 AtomUI `Breadcrumb`，路径从 `{bucket}` 开始，不显示额外 `oss` 根文案。
- 搜索为 OSS provider 原生左前缀搜索，不是本地搜索；搜索结果继续遵守分页。
- OSS 文件列表支持上一页、下一页。
- OSS bucket 列表页右下角 FloatButton 为刷新和新建 bucket；具体 bucket/path 页为刷新、上传、新建文件夹。
- 第一版禁用 OSS 文件夹重命名和文件夹删除；文件删除可用。

## 5. FTP/SFTP 文件浏览

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ [provider name]                                                              │
│ [/home/deploy/folder breadcrumb]                                             │
├────────────┬───────────────────────────────┬─────────────┬───────────────────┤
│ 缩略图     │ 文件名                        │ 文件大小    │ 修改时间          │
├────────────┼───────────────────────────────┼─────────────┼───────────────────┤
│ [folder]   │ folder-a                      │ -           │ 2026-06-13        │
│ [file]     │ file.zip                      │ 1.2 GB      │ 2026-06-13        │
├────────────┴───────────────────────────────┴─────────────┴───────────────────┤
│ 当前目录项目 2 个                                                            │
│                                                                   [刷新]     │
│                                                                   [上传]     │
│                                                                   [新建夹]   │
└──────────────────────────────────────────────────────────────────────────────┘
```

FTP/SFTP 规则：

- 使用 AtomUI `Breadcrumb` 展示路径，并支持点击 breadcrumb 节点返回对应目录。
- SFTP 账号测试成功后可回填 Home；打开账号时默认进入 Home。
- FTP/SFTP 协议原生不提供统一分页语义，第一版不显示上一页、下一页。
- 第一版不提供 FTP/SFTP 服务端搜索框。

## 6. 文件列表

第一版文件列表采用行列表，不做网格视图。

固定四列：

| 列 | 内容 | 第一版规则 |
| --- | --- | --- |
| 缩略图 | 固定文件图标或文件夹图标 | 第一版使用 AtomUI 内置 AntDesign 图标展示文件类型，不生成真实内容缩略图。 |
| 文件名 / 桶名 | 远程对象名称 | bucket 列表时显示“桶名”，其他列表显示“文件名”。 |
| 文件大小 | 文件大小 | 文件夹和 bucket 显示 `-`。 |
| 修改时间 | 远程对象修改时间 | 统一使用“修改时间”。 |

## 7. 行交互

| 行为 | 规则 |
| --- | --- |
| 鼠标悬停 | 当前行背景变色。 |
| 右键行 | 先选中该行；右键菜单响应范围为整行。 |
| 双击 bucket | 进入该 bucket，并同步 bucket ComboBox。 |
| 双击文件夹 | 进入该目录并刷新文件列表。 |
| 双击文件 | 第一版不定义默认打开行为。 |

第一版只要求单选。多选、框选、快捷键选择后续再设计，不进入本文档。

## 8. 右键菜单

右键菜单第一版按对象类型展示：

| 对象类型 | 右键菜单 |
| --- | --- |
| 文件 | 预览、下载、重命名、删除 |
| 文件夹 | 打开、重命名、删除 |
| OSS bucket | 不提供右键操作 |

特殊规则：

- OSS 文件夹的重命名和删除在第一版禁用。
- 删除必须弹出确认弹窗。
- 重命名、删除、新建文件夹成功后优先根据操作结果更新当前 UI 列表，避免不必要的整页刷新。
- 上传可一次选择多个本地文件；创建成功后任务应一次性进入传输队列，由 Transfer 并发池按设置执行。
- 下载创建成功后跳转传输队列并提示。
- 预览只对支持格式内的图片和小文本文件开放，不进入传输队列，不写入传输历史。

## 8.1 文件预览

远程文件预览用于快速查看图片和小文本内容。

支持图片格式：

- `.png`
- `.jpg`
- `.jpeg`
- `.bmp`
- `.gif`
- `.webp`

支持文本格式：

- `.txt`
- `.log`
- `.md`
- `.markdown`
- `.json`
- `.xml`
- `.yaml`
- `.yml`
- `.csv`
- `.ini`
- `.conf`
- `.config`
- `.html`
- `.css`
- `.js`
- `.ts`
- `.cs`
- `.py`
- `.java`
- `.go`
- `.rs`
- `.cpp`
- `.c`
- `.h`
- `.sql`
- `.sh`
- `.ps1`

预览规则：

- 文本最大 `1 MB`。
- 图片最大 `10 MB`。
- 文本编码支持 UTF-8、UTF-8 BOM、UTF-16 LE、UTF-16 BE。
- bucket、文件夹、未知格式、超限文件不开放可用预览动作。
- 图片预览使用 AtomUI `ImagePreviewer`，标题栏只显示文件名，并提供加载、失败、缩放、旋转、翻转、拖动和适应窗口能力。
- ImagePreviewer 通过 `StreamImagePreviewSource` 按需调用 Application 读取远程图片；关闭预览时取消未完成读取并释放 Stream。
- 文本预览弹窗展示只读文本、文件名、大小和编码。
- 预览失败时复用统一错误提示或错误详情入口。
- 预览必须通过 Application 用例进入系统，不能由 ViewModel 直接调用 Provider。

## 9. 加载、反馈和 FloatButton

- 网络请求期间使用 AtomUI `Spin` 展示加载遮罩。
- 加载期间 FloatButton 不可点击，视觉上应被遮罩覆盖或隐藏。
- 文件列表页右下角操作使用 AtomUI `FloatButton`，不使用右上角文字按钮。
- 操作成功和失败通过统一消息服务展示；批量操作通知应合并，避免一次批量事件弹出大量消息。

## 10. 分页器

- OSS 文件浏览保留上一页、下一页。
- FTP/SFTP 不显示分页器。
- 分页状态属于当前远程浏览页 ViewModel。
- 分页器不进入左侧菜单，不作为路由节点。

## 11. 页面状态

远程浏览页第一版至少需要以下状态：

| 状态 | 展示 |
| --- | --- |
| 加载中 | 文件列表区域显示 AtomUI Spin。 |
| 空目录 | 文件列表区域显示空状态。 |
| 加载失败 | 文件列表区域显示错误摘要，并提供重试和详情入口。 |
| 汇总首页 | 展示所有账号的分组入口和统计。 |
| 分类账号列表 | 展示某一分类的连接选择列表。 |
| 无账号空状态 | 展示 AtomUI Empty 和添加账号按钮。 |

错误详情通过弹窗展示，不直接塞进文件列表行。

## 12. 设计约束

- 只有具体连接实例才能直接触发文件列表加载。
- 远程存储总入口只展示账号汇总，不请求远程文件列表。
- 无账号空状态的新增账号入口必须复用统一账号弹窗。
- 左侧菜单不展示远程目录树。
- 文件列表第一版只做行列表，缩略图列只展示 AtomUI 内置类型图标，不做真实内容缩略图，不做多选。
- 删除必须二次确认。
- 上传、下载、删除、重命名、新建文件夹必须通过 Application 用例进入系统，不能由 ViewModel 直接调用 Provider。

## 13. AtomUI 使用

| 场景 | AtomUI 组件 |
| --- | --- |
| 汇总统计 | `Statistic` |
| 汇总分组 | `Card` |
| provider 类型标签 | `Tag` |
| 空状态 | `Empty` |
| 加载中 | `Spin` |
| 浮动操作 | `FloatButton` |
| 路径导航 | `Breadcrumb` |
| bucket 选择 | `ComboBox` |
| 搜索输入 | `LineEdit` + `Button` |
| 确认弹窗 | `Modal` |
| 文件类型图标 | AtomUI AntDesign 图标 |
| 图片预览 | `ImagePreviewer` + `StreamImagePreviewSource` |
