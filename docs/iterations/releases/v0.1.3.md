# AtomBox v0.1.3 迭代计划

> 文档状态：Draft
>
> 创建时间：2026-07-05
>
> 范围：记录 v0.1.3 增强型功能范围、实现顺序、测试底线和发布前验收清单。

## 版本目标

v0.1.3 聚焦远程浏览页轻量预览能力，允许用户直接预览图片和小文本文件。

本版本不引入新的远程存储 Provider，不修改 Provider 契约，不让预览进入 Transfer 队列，也不把预览行为写入传输历史。

## 本版本范围

| 功能 | 状态 | 设计文档 |
|---|---|---|
| 远程图片与小文本文件预览 | In Progress | `../features/remote-file-preview.md` |

本版本包含：

- 远程图片预览。
- 远程小文本预览。
- Core 预览模型。
- Application 预览用例。
- Desktop 远程浏览页预览入口和弹窗。
- 相关单元测试、手工 smoke test 和正式文档同步。

本版本不包含：

- Office 文档预览。
- PDF、音频、视频预览。
- 系统默认应用打开。
- 真实远程文件缩略图生成。
- Provider 契约调整。
- Transfer 队列、历史或任务语义调整。

## 实现顺序

1. 同步正式设计文档：Core 模型、Application 用例、远程浏览页 UI。
2. 实现 Core 预览模型和默认限制。
3. 实现 Application 预览用例，复用现有 provider `DownloadAsync(..., Stream, ...)` 读取到内存流。
4. 实现 Desktop 预览入口、图片弹窗、文本弹窗、加载状态和错误提示。
5. 补充 Core / Application 单元测试。
6. 执行 Desktop 手工 smoke test。
7. 将 `../features/remote-file-preview.md` 状态更新为 `Done`，并补充实际验收结果。

## 测试底线

- Core / Application 单元测试必须覆盖预览类型识别、大小限制、provider 成功、provider 失败和实际读取超限。
- Desktop 至少完成手工 smoke test，覆盖图片、小文本、不支持格式、超大文件、bucket、文件夹。
- 本版本不要求真实云 provider opt-in 测试作为阻塞项；如已有真实账号可用，可以补充非阻塞 smoke 记录。
- 预览行为不得创建 Transfer 任务，不得进入传输队列，不得写入传输历史。
- Provider 契约不得新增 `DownloadToMemoryAsync`、`DownloadForPreviewAsync` 等场景化方法。

## 版本验收标准

- 远程浏览页文件行可触发“预览”动作。
- 支持格式内的图片文件可以打开图片预览弹窗。
- 支持格式内的小文本文件可以打开只读文本预览弹窗。
- bucket、文件夹不开放可用预览动作。
- 不支持格式提示清晰，并引导用户下载后查看。
- 超过大小限制的文件提示清晰，并引导用户下载后查看。
- 预览失败时保留统一错误提示或错误详情入口。
- 预览功能不影响现有列表、上传、下载、删除、重命名和新建文件夹行为。
- Windows 桌面 smoke test 通过。
