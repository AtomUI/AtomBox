# AtomBox v0.1.6 迭代计划

> 文档状态：Draft
>
> 创建时间：2026-07-21
>
> 范围：在 Desktop UI 中优先使用 AtomUI 提供的视觉组件，统一交互与视觉表现。

## 版本目标

v0.1.6 聚焦桌面端视觉组件收敛。对 AtomBox 已有页面逐项检查，在不改变现有业务流程和模块边界的前提下，使用 AtomUI.Desktop.Controls 提供的成熟组件替换重复或手工拼装的基础视觉控件，减少自定义样式和交互代码。

本版本不是页面重设计，也不以机械替换全部 Avalonia 控件为目标。布局容器、普通文本、启动错误窗口以及不符合 AtomBox 数据模型的组件继续保留现有实现。

## 本版本范围

| 功能 | 状态 | 设计文档 |
|---|---|---|
| Desktop AtomUI 视觉组件迁移 | Implemented | `../features/atomui-visual-components.md` |

本版本计划包含：

- 远程文件浏览器搜索框迁移为 AtomUI `SearchEdit`。
- 页面内可替换的原生 Avalonia `Button` 迁移为 AtomUI `Button`，并优先使用 AtomUI AntDesign 图标。
- 传输队列、传输历史、首页最近传输和远程文件列表的空状态迁移为 AtomUI `Empty`。
- 左侧“传输队列”数量提示迁移为 AtomUI `CountBadge`。
- 通用确认弹窗优先迁移为 AtomUI `MessageBox`。
- 错误详情展示评估并迁移为 AtomUI `Descriptions`。
- 传输进度展示评估并迁移为 AtomUI `ProgressBar`，同时保留必要的状态文案。
- 账号类型、Provider 类型或传输结果等短状态信息按适用场景使用 AtomUI `Tag`。
- 远程浏览错误区域按适用场景使用 AtomUI `Alert`，保留重试等操作入口。
- 设置页面的手工分隔线按适用场景使用 AtomUI `Separator`。
- 补充构建、自动化测试和 Desktop 手工视觉验收。

## 非目标

- 不修改 Core、Application、Providers、Transfer 或 Infrastructure 的业务契约。
- 不改变上传、下载、预览、删除、账号管理和传输调度流程。
- 不进行整体页面布局重设计或主题系统重构。
- 不机械替换 `Grid`、`Border`、`StackPanel`、普通 `TextBlock` 等基础布局和展示元素。
- 不替换 `StartupErrorWindow` 的原生 Avalonia 控件；该窗口必须在 AtomUI 初始化失败时仍可显示。
- 不把游标分页或无可靠总数的分页强行迁移为 AtomUI `Pagination`。
- 不使用 `MarqueeLabel` 替换文件名截断与 Tooltip。
- 不使用语义不匹配的 AtomUI Upload/Transfer 控件替代 AtomBox 传输业务。
- 不默认引入 `AtomUI.Desktop.Controls.DataGrid`；表格迁移须经独立评估后才能纳入本版本。

## 实现顺序

1. 建立现有控件、目标 AtomUI 控件和受影响页面的迁移清单。
2. 先迁移低风险组件：`SearchEdit`、`Button`、`Empty` 和 `CountBadge`。
3. 再迁移弹窗与信息展示：`MessageBox`、`Descriptions`、`Tag`、`Alert` 和 `Separator`。
4. 在数据模型足够支持时迁移传输 `ProgressBar`，不制造虚假的进度信息。
5. 对键盘操作、焦点、禁用态、加载态、空状态和窄窗口布局执行回归检查。
6. 完成构建、自动化测试和 Desktop 手工 smoke test，并同步正式 UI 文档。
7. 独立评估 DataGrid 的 NuGet 依赖、列宽、选择、右键菜单和虚拟化能力，再决定是否纳入 v0.1.6。

## 版本验收标准

- 设计文档列出的低风险替换项完成，或明确记录未替换原因。
- 替换后的组件遵循 AtomUI 主题、尺寸、焦点和禁用态表现。
- 原有命令、绑定、校验、提示、菜单徽标和传输状态行为没有回归。
- 空状态、错误状态和操作按钮在常见桌面窗口尺寸下不重叠、不溢出。
- 不因视觉迁移引入跨层业务依赖。
- `StartupErrorWindow` 在不依赖 AtomUI 正常初始化的条件下仍可使用。
- 完整构建和自动化测试通过。
- 关键页面完成手工视觉与交互 smoke test。
