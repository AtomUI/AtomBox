# AtomBox v0.1.5 迭代计划

> 文档状态：Draft
>
> 创建时间：2026-07-14
>
> 范围：远程图片预览改用 AtomUI ImagePreviewer。

## 版本目标

v0.1.5 只包含一项功能：使用 AtomUI.Desktop.Controls 6.0.8 的 `ImagePreviewer` 和 `StreamImagePreviewSource` 替换当前远程图片的基础预览界面，补齐专业图片查看能力和加载生命周期。

## 本版本范围

| 功能 | 状态 | 设计文档 |
|---|---|---|
| AtomUI ImagePreviewer 图片预览 | Done | `../features/atomui-image-previewer.md` |

本版本包含：

- Core 图片预览元数据和 Stream 结果契约调整。
- Application 图片预检与按需读取 Stream 用例。
- Desktop ImagePreviewer 宿主、StreamImagePreviewSource 和生命周期管理。
- 图片加载、失败、缩放、旋转、翻转、拖动和适应窗口交互。
- 自动化测试、Desktop 手工验收和正式文档同步。

本版本不包含：

- Providers、Transfer 或 Infrastructure 改动。
- 文本预览调整。
- 图片缓存、临时文件、Range 请求、渐进式解码和多图预览。
- 支持格式或预览大小上限调整。

## 实现顺序

1. 落地 v0.1.5 release 与 feature 设计文档。
2. 调整 Core 图片预览结果并新增 Stream 结果模型。
3. 调整 Application 图片预检，新增按需打开图片预览 Stream 用例。
4. 在 Desktop 主窗口挂载专用 ImagePreviewer，由 DialogService 管理打开、关闭和清理。
5. 补充 Application 自动化测试并完成 Desktop 手工 smoke test。
6. 同步 Core、Application 和远程浏览 UI 正式文档。
7. 执行完整构建与测试，将 feature 状态更新为 Done 并记录结果。

## 版本验收标准

- 图片预览使用 AtomUI ImagePreviewer，不再使用 Avalonia `Image + Bitmap + ScrollViewer`。
- 图片窗口先打开并展示加载状态，再按需读取远程内容。
- 标题栏只显示文件名。
- 图片缩放、旋转、翻转、拖动和适应窗口可用。
- 关闭预览能够取消未完成读取并释放 Stream。
- 图片仍受 10 MB 上限保护，超限和 Provider 失败进入清晰的失败状态。
- 文本预览无回归。
- 预览不创建 Transfer 任务，不写入队列、历史或本地缓存。
- Providers、Transfer、Infrastructure 没有代码改动。
- 完整构建和测试通过。
