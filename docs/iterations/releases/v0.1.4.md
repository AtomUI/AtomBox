# AtomBox v0.1.4 迭代计划

> 文档状态：Draft
>
> 创建时间：2026-07-07
>
> 范围：记录 v0.1.4 本地文件指纹索引能力范围、实现边界、上传提示行为、传输队列入口提示和发布前验收清单。

## 版本目标

v0.1.4 聚焦本地文件指纹索引，为上传前快速判断“当前存储账号是否曾经上传过相同文件”提供基础能力；同时优化上传后的页面跳转体验，避免小文件上传完成后用户被带到空的传输队列页面。

本版本只做本地索引，适用于所有支持上传的 Provider，不限于 OSS。本版本不做跨设备同步，不承诺对象存储侧的“秒传”，也不要求各家 OSS 提供按 MD5 查询已上传文件的能力。

## 本版本范围

| 功能 | 状态 | 设计文档 |
|---|---|---|
| 本地文件指纹索引 | Done | `../features/local-file-fingerprint-index.md` |
| 上传后留在当前页面与传输队列数量提示 | Done | 本文 |

本版本包含：

- 本地文件指纹索引模型。
- 本地 JSON 索引文件持久化。
- 上传前按 `sha256 + fileSize + storageAccountId` 查询本地索引的 Application 编排点。
- 命中历史记录时返回具体远端路径，供 Desktop 提示用户是否再次上传。
- 上传任务携带上传前计算出的指纹元数据。
- 上传成功后由 Infrastructure 的 `ITransferStateStore` 装饰器写入或更新本地索引。
- 设置页提供“指纹索引”统一开关；索引信息和清空入口暂不显示。
- 远程文件浏览器底部展示上传前指纹计算状态。
- 上传任务创建后不自动跳转到“传输队列”，用户停留在当前远程文件浏览器页面。
- 上传任务创建过程和创建结果通过顶部 Message 提示。
- 左侧“传输队列”菜单旁展示传输任务数量，上传任务创建成功后立即显示本次创建数量，随后由真实队列刷新覆盖；数量为 0 时隐藏。
- 传输任务数量大于 0 时自动展开左侧“传输”父菜单，确保用户能看到“传输队列”的数字变化。
- Infrastructure 基于 AtomBox 应用数据目录管理索引文件。
- 单元测试和本地持久化测试。

本版本不包含：

- SQLite。
- JSONL。
- 跨设备同步。
- 远端索引文件。
- AtomBox 自建云同步服务。
- 跨 Provider、跨账号的纯内容级命中提示。
- 依赖 OSS ETag 作为文件 MD5。
- 基于 MD5 的对象存储服务端秒传承诺。
- Provider 契约调整。
- 传输队列 UI 调整。
- 新增 `TransferStatus`。
- 上传准备阶段进入 Transfer worker。
- 让 Transfer worker 直接依赖指纹索引端口或 JSON 实现。
- 指纹计算进度条或取消按钮。
- 多个上传准备会话并发运行。
- 下载任务创建后的页面跳转行为调整。

## 实现要点

1. 不使用 SQLite，第一版使用普通 JSON 文件。
2. 索引文件放在 AtomBox 应用数据目录，由现有 `AtomBoxStoragePaths` 管理。
3. Core 定义模型和端口，包括指纹索引端口以及 `TransferTask` 可选指纹元数据。
4. Application 负责上传前指纹计算、账号范围查询、用户确认，并创建带指纹元数据的上传任务。
5. Infrastructure 负责 JSON 索引持久化，并通过 `FingerprintAwareTransferStateStoreDecorator` 在上传成功后写入索引。
6. `FingerprintAwareTransferStateStoreDecorator` 通过 `IStorageAccountRepository` 根据 `storageAccountId` 解析 `providerId` 等索引记录所需的 Provider 信息。
7. Transfer worker 仍只依赖 `ITransferStateStore`，不感知指纹索引。
8. 查询语义为账号范围命中：`sha256 + fileSize + storageAccountId`。
9. 查询命中后由 Desktop 弹窗展示历史远端路径，并提供“取消”和“再次上传”动作。
10. 设置页使用 `atom:ToggleSwitch` 管理是否启用上传指纹索引，默认关闭，UI 文案为“指纹索引”。
11. 设置页当前只展示开关；索引文件路径、记录数量、最近更新时间和清空全部索引入口暂不显示。
12. 上传前指纹计算 UI 放在远程文件浏览器底部 `StatusMessage` 区域。
13. 指纹计算期间禁用上传按钮；同一远程文件浏览器同一时刻只允许一个上传准备会话。
14. 上传任务创建后不调用 `NavigationTarget.TransferQueue`，只更新当前页状态并通过顶部 Message 告知用户。
15. 左侧导航菜单的“传输队列”节点使用 AtomUI `NavMenuNode.HeaderTemplate` 自定义标题布局，并在文案右侧展示数字徽标。
16. 上传任务创建成功后由 `RemoteBrowserViewModel` 通知 `StatusBarViewModel` 立即更新徽标数量；之后复用状态栏已有的传输队列轮询结果覆盖真实数量，不为菜单徽标新增独立轮询。
17. 当传输任务数量变为大于 0 时，如果左侧导航的“传输”父菜单未展开，Shell 自动展开该父菜单。

推荐文件位置：

```text
{AtomBoxStoragePaths.StateDirectory}/fingerprints/file-fingerprint-index.json
```

Windows 默认路径形态：

```text
%APPDATA%/AtomBox/state/fingerprints/file-fingerprint-index.json
```

## 实现顺序

1. 同步正式设计文档：Core 模型和端口、Application 上传用例、Infrastructure 本地状态存储。
2. 在 Core 定义文件指纹记录、查询请求、索引端口和 `TransferTask` 可选指纹元数据。
3. 在 Infrastructure 增加 JSON 指纹索引实现，路径接入 `AtomBoxStoragePaths`。
4. 在 Infrastructure 增加 `FingerprintAwareTransferStateStoreDecorator`，包装真实 `TransferStateStore`。
5. 调整 DI：注册真实 `TransferStateStore`，注册 `IFileFingerprintIndexStore`，并将 `ITransferStateStore` 绑定为装饰器。
6. 在 Application 上传编排中加入上传前账号范围查询，并创建带指纹元数据的上传任务。
7. 在 Settings 页面接入“指纹索引”开关。
8. 在 Remote Browser 上传入口接入底部指纹计算文案、上传按钮禁用和历史命中提示弹窗。
9. 调整 Remote Browser 上传入口：创建上传任务后留在当前页并显示顶部 Message。
10. 调整左侧传输菜单：为“传输队列”增加右侧数字徽标。
11. 补充 Core / Application / Infrastructure / Desktop 测试。
12. 执行上传主路径 smoke test。
13. 将 `../features/local-file-fingerprint-index.md` 状态更新为 `Done`，并补充实际验收结果。

## 测试底线

- Core 测试覆盖指纹记录基本校验、`TransferTask` 可选指纹元数据和 `sha256 + fileSize + storageAccountId` 查询匹配规则。
- Infrastructure 测试覆盖 JSON 文件首次创建、读取、追加/更新、统计、清空全部、损坏文件处理、路径创建、传输状态存储装饰器写入规则和账号解析失败不写入规则。
- Application 测试覆盖设置关闭时跳过指纹逻辑、上传前同账号查询命中、未命中、用户取消、用户选择再次上传、创建带指纹元数据的上传任务。
- Desktop 测试或手工验证覆盖设置页开关、远程文件浏览器底部计算文案、上传按钮禁用、历史命中弹窗、“取消”和“再次上传”动作。
- 不要求真实云 provider opt-in 测试作为阻塞项；如已有真实账号可用，可以补充非阻塞 smoke 记录。
- 不能把索引文件写入项目目录、安装目录或用户下载目录。

## 版本验收标准

- 本地索引文件创建在 AtomBox 应用数据目录下。
- 第一版索引格式为普通 JSON 文件。
- 设置页可以开启或关闭上传指纹索引。
- 新安装或重置设置后，“指纹索引”默认关闭。
- 设置开启且用户确认上传时，正式上传任务携带完整指纹元数据。
- 上传成功后能记录本地文件指纹和远端目标信息。
- 上传失败或取消时不记录成功索引。
- 远程文件浏览器上传前能在底部显示指纹计算状态。
- 单文件显示 `正在计算文件指纹：demo.txt`；多文件显示 `正在计算文件指纹 1/3：demo.txt`。
- 指纹计算期间上传按钮禁用，第二批上传准备请求被阻止并提示稍后再试。
- 后续在同一 `storageAccountId` 下上传同一 `sha256 + fileSize` 文件时，Application 能查询到历史记录。
- 历史命中提示能展示上一次或多次上传到的具体远端路径。
- 用户可以在历史命中提示中取消上传，也可以选择再次上传。
- 选择文件上传后页面不自动跳转到“传输队列”。
- 上传任务创建中和创建成功后，当前页顶部有 Message 提示。
- “传输队列”菜单旁显示传输任务数量；选择 2 个文件并成功创建上传任务后应立即显示 `2`，队列为空时不显示数字。
- 创建上传任务后，“传输”父菜单应自动展开，用户无需手动展开即可看到“传输队列”数量。
- 下载任务创建后的自动跳转行为保持不变。
- 索引逻辑不修改 Provider 契约，不进入 Transfer 内部 worker 细节。
- Transfer worker 不直接依赖指纹索引端口或 JSON 实现。
- 上传成功后的索引写入通过 `FingerprintAwareTransferStateStoreDecorator` 完成。
- 装饰器通过 `IStorageAccountRepository` 解析索引记录所需的 Provider 信息。
- 指纹索引写入失败不得把已成功上传的任务改成失败。
- 传输队列 UI 和 `TransferStatus` 保持不变。
- 索引文件不得包含 secret material。
