# AtomBox v0.1.4 迭代计划

> 文档状态：Draft
>
> 创建时间：2026-07-07
>
> 范围：记录 v0.1.4 本地文件指纹索引能力范围、实现边界、上传提示行为和发布前验收清单。

## 版本目标

v0.1.4 聚焦本地文件指纹索引，为上传前快速判断“当前存储账号是否曾经上传过相同文件”提供基础能力。

本版本只做本地索引，适用于所有支持上传的 Provider，不限于 OSS。本版本不做跨设备同步，不承诺对象存储侧的“秒传”，也不要求各家 OSS 提供按 MD5 查询已上传文件的能力。

## 本版本范围

| 功能 | 状态 | 设计文档 |
|---|---|---|
| 本地文件指纹索引 | Planned | `../features/local-file-fingerprint-index.md` |

本版本包含：

- 本地文件指纹索引模型。
- 本地 JSON 索引文件持久化。
- 上传前按 `sha256 + fileSize + storageAccountId` 查询本地索引的 Application 编排点。
- 命中历史记录时返回具体远端路径，供 Desktop 提示用户是否再次上传。
- 上传成功后写入或更新本地索引的 Application 编排点。
- Infrastructure 基于 AtomBox 应用数据目录管理索引文件。
- 单元测试和本地持久化测试。

本版本不包含：

- SQLite。
- JSONL。
- 跨设备同步。
- 远端索引文件。
- AtomBox 自建云同步服务。
- 跨 Provider、跨账号的纯内容级命中提示。
- 依赖 OSS ETag 作为文件 MD5。
- 基于 MD5 的对象存储服务端秒传承诺。
- Provider 契约调整。

## 实现要点

1. 不使用 SQLite，第一版使用普通 JSON 文件。
2. 索引文件放在 AtomBox 应用数据目录，由现有 `AtomBoxStoragePaths` 管理。
3. Core 定义模型和端口，Infrastructure 负责持久化，Application 在上传前后编排查询和写入。
4. 查询语义为账号范围命中：`sha256 + fileSize + storageAccountId`。
5. 查询命中后由 Desktop 弹窗展示历史远端路径，并提供“取消”和“再次上传”动作。

推荐文件位置：

```text
{AtomBoxStoragePaths.StateDirectory}/fingerprints/file-fingerprint-index.json
```

Windows 默认路径形态：

```text
%APPDATA%/AtomBox/state/fingerprints/file-fingerprint-index.json
```

## 实现顺序

1. 同步正式设计文档：Core 模型和端口、Application 上传用例、Infrastructure 本地状态存储。
2. 在 Core 定义文件指纹记录、查询请求和索引端口。
3. 在 Infrastructure 增加 JSON 文件实现，路径接入 `AtomBoxStoragePaths`。
4. 在 Application 上传编排中加入上传前账号范围查询和上传成功后写入。
5. 在 Desktop 上传入口接入历史命中提示弹窗。
6. 补充 Core / Application / Infrastructure / Desktop 测试。
7. 执行上传主路径 smoke test。
8. 将 `../features/local-file-fingerprint-index.md` 状态更新为 `Done`，并补充实际验收结果。

## 测试底线

- Core 测试覆盖指纹记录基本校验和 `sha256 + fileSize + storageAccountId` 查询匹配规则。
- Infrastructure 测试覆盖 JSON 文件首次创建、读取、追加/更新、损坏文件处理和路径创建。
- Application 测试覆盖上传前同账号查询命中、未命中、用户取消、用户选择再次上传、上传成功写入、上传失败不写入。
- Desktop 测试或手工验证覆盖历史命中弹窗、历史远端路径展示、“取消”和“再次上传”动作。
- 不要求真实云 provider opt-in 测试作为阻塞项；如已有真实账号可用，可以补充非阻塞 smoke 记录。
- 不能把索引文件写入项目目录、安装目录或用户下载目录。

## 版本验收标准

- 本地索引文件创建在 AtomBox 应用数据目录下。
- 第一版索引格式为普通 JSON 文件。
- 上传成功后能记录本地文件指纹和远端目标信息。
- 上传失败或取消时不记录成功索引。
- 后续在同一 `storageAccountId` 下上传同一 `sha256 + fileSize` 文件时，Application 能查询到历史记录。
- 历史命中提示能展示上一次或多次上传到的具体远端路径。
- 用户可以在历史命中提示中取消上传，也可以选择再次上传。
- 索引逻辑不修改 Provider 契约，不进入 Transfer 内部 worker 细节。
- 索引文件不得包含 secret material。
