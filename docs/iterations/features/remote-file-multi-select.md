# 远程文件搜索与批量操作

> 状态：Implemented / UI Disabled
>
> 目标版本：v0.1.7
>
> 创建时间：2026-07-23
>
> 影响模块：Desktop / Application

## 背景

远程文件浏览器需要完善键盘搜索、搜索清空和下载反馈。批量操作能力也已实现，但当前版本暂不开放对应 UI。

## 目标

- `SearchEdit` 获得焦点时，按 Enter 执行与搜索按钮相同的查询。
- 清空已执行的搜索后，自动恢复当前目录完整列表。
- 单文件下载任务创建后保持当前页面，通过顶部 Message 和传输队列数字反馈。
- 文件列表增加明确的多选入口、当前页全选和选择数量反馈。
- 支持对选中文件批量下载或删除。
- 在 OSS、FTP/SFTP 和 WebDAV 中保持一致交互。

## 用户体验

- v0.1.7 默认关闭批量选择功能开关，不展示以下多选 UI。
- 每个文件行显示复选框；文件夹和 Bucket 不参与批量选择。
- 表头复选框控制当前已加载页面中的全部文件，并支持未选、部分选择和全选状态。
- 选中至少一个文件后，在列表顶部显示“已选择 N 个文件”、下载、删除和取消选择。
- 单文件双击预览、文件夹双击进入以及现有右键菜单保持不变。
- 刷新、搜索、切换账号、切换 Bucket、进入目录和翻页时清空选择。
- 批量下载直接写入默认下载目录，不自动跳转传输队列；顶部消息提示创建结果。
- 批量删除只弹出一次确认框。执行时逐个删除，完成后刷新列表并提示成功数和失败数；部分失败时提供错误详情。

## 设计方案

### Desktop

- 使用单一内部功能开关控制选择列宽和批量操作栏可见性，默认关闭。
- 关闭时选择列宽为 `0`，复选框不可见且不可交互，批量操作栏不会显示。
- `RemoteListRowViewModel` 增加可变的 `IsSelected` 状态，并由页面 ViewModel 汇总选中数量。
- 新增当前页全选状态、批量下载、批量删除和清空选择命令。
- 选择集合以当前 `Rows` 为唯一来源，不跨页面持久化。
- 搜索框通过键盘事件调用现有 `ApplySearchAsync`，避免复制搜索逻辑。
- 搜索框输入值与已应用查询分开保存；清空已应用查询时重置分页并恢复完整列表。
- 单文件下载复用现有任务创建和队列计数机制，不自动切换页面。
- 批量删除复用 `RemoteBrowserAppService.DeleteRemoteItemAsync`，按顺序执行并收集失败项。

### Application

- 下载请求由“多个远程路径共享一个本地路径”调整为显式目标列表。
- 每个下载目标包含一个 `RemotePath` 和一个 `LocalPath`，每个目标生成独立传输任务。
- 保留现有覆盖策略、调度器和传输执行流程。

### 模块边界

- Core 的传输任务模型保持不变。
- Providers 继续提供单对象下载和删除能力。
- Transfer 继续按独立任务执行，不感知 UI 批量选择。
- Infrastructure 的本地文件写入和任务持久化保持不变。

## 失败与边界

- 没有可选文件时，批量操作按钮不可用。
- 同名文件使用现有 `Rename` 策略，避免覆盖本地文件。
- 批量删除中的单项失败不阻止后续项执行。
- 加载期间禁用选择和批量操作，避免列表变化时操作过期路径。
- Enter 搜索在正在加载时不重复发起请求。
- 加载期间清空搜索时，当前加载结束后再恢复完整列表，避免并发重复请求。

## 测试与验收

- Application 测试覆盖多个下载目标生成独立任务及各目标路径映射。
- Application 验证测试覆盖空目标列表、空远程路径和空本地路径。
- Desktop 手工检查单选、多选、当前页全选、部分选择、清空选择。
- 分别在 OSS、FTP/SFTP 和 WebDAV 检查批量下载和批量删除。
- 检查搜索 Enter、搜索按钮、清除按钮、手工清空、空关键字和加载状态。
- 检查单文件下载后页面保持不变，顶部 Message 和传输队列数字正确更新。
- 检查目录切换、刷新、翻页和账号切换后选择被清空。
- 默认关闭状态下检查列表没有选择列、复选框、批量操作栏或多余空白。
- 临时打开内部开关时检查原有多选 UI 能够完整恢复；提交代码保持关闭。
