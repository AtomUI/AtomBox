## 9. Phase 7：Transfer 最小闭环

目标：打通传输任务从创建到状态展示的最小链路。

允许做：

- 创建传输任务模型落库。
- 创建传输队列。
- 创建最小调度器。
- 创建单 worker 执行路径。
- 创建进度快照。
- 创建任务状态更新。
- 创建队列页和历史页的数据读取路径。

禁止做：

- 一开始做复杂并发调度。
- 一开始做断点续传。
- 一开始做复杂限速策略。
- 一开始做自动恢复历史任务。
- 让 UI 直接读取 Transfer 内部队列。

完成条件：

- 用户可以创建一个传输任务。
- Transfer Runtime 可以执行或模拟执行任务。
- 任务状态可以持久化。
- UI 可以通过 Application 查询队列和历史快照。
